初探响应式布局

发表于 2018-09-30 | 更新于 2019-04-12 | 分类于 前端小零件 | 评论数：

        github：https://github.com/GEPANG/responsive

       光阴似箭，不知不觉，暑假已经收假一个月了。在这段时间里我继续跟进关于前端小知识———响应式布
　　局。
       响应式布局也就是能自适应屏幕，即在pc端，移动端都能够适应并且页面清楚。当然刚开写这个没有用到框
   架，是直接写的。在开始是以弹性为基础写的，但是弹性布局在较宽的屏幕和较窄的屏幕适应不好，因为弹性布
   局的优化是有限的。所有后面就提出了向响应式发展的理念——也就是给不同的设备做不同设计。在不同层面来
   处理，这里的不同层面来处理也就是响应式网站设计，这时候就引入了媒体查询也叫媒介查询(media query)，
   这个概念在css2.1就开始有了，在css3以后这个媒体查询的概念就越来越被强化了。了解更多请往下看……

        媒体查询

        css2的媒体查询

        css3媒体查询

        这就是一个典型的媒体查询以@media开头，all表示媒体类型，也可以是screen和print，当然也可以不写
    默认为all，min-width:800px表示最小宽度为800px；and表示逻辑操作的连接，也可以是only和not，or,当然
    也可以是逗号，逗号和or一样，这些我就不详细说了。

        一个不如意的响应式布局

        一、写静态网页

    在这部分也没有多少技巧，相信来写这个响应式的，对于写个静态网页应是没有问题的。

    看一下注意事项：

  1. 需要用百分比来写这个网页，否则网页不会改变。
  2. 页面重置(初始化)，html{font-size: 62.5%;color: #222;}；这部分代码是吧页面文字大小设置为10px即1rem
    =10px{(10/16)*100%}。当html{font-size: 100%;color: #222;}是1rem=16px;
  3. 单位使用，在相应式布局中一般都用到em和rem。为什么要用两个呢？原因是相应式要的是相对单位，因为
    在写页面时，部分内容会随着页面宽度变化而发生改变。所以用em和rem这两个相对单位。
    (1)、而相对单位em，既然是相对就会相对于谁也就是参照物，em的参照物为父元素的font-size的大小，具
    有继承性，1em=16px；由于em随着父元素给写网页带来了许多不确定性，因此会容易导致混乱。
    (2)、而相对单位rem。它基本是和em是一样的，但是其参照物为根元素html，相对有参考固定不变的值，这
    样就好计算，不会混乱。因此一般都是用rem来写。
  4. 图片宽度要设置为100%，这个在后面再讲解图片响应式。

        二、写响应式

        静态的写网了，就开始写真正要做的事情了——响应式。
        在开始讲过，响应式就是用媒体查询来写，来适应任何分辨率的屏幕。我在写响应式的时候，没有响应式
    的psd图，都是以自己的想法来写这个响应式，以调整屏幕来写的。

        需要注意的：当在写这个@media all and (min-width:50em){...};媒体查询的时候，在后面设置断点的单
    位时不能写成rem，原因是——媒体查询的优先级特别高，它并不是html的一个子元素，而并不在html级别之
    下，所以它不是html里font-size的设置而是浏览器的默认设置，因此可以看到这里写的50em即50*16=800px。


        看一下这段代码:

    @media only screen and (max-width:75em){
        header .menu .one .first:nth-child(4),
        header .menu .one .first:nth-child(6)
    {
        display: none;
    }
        header .menu .one{
        font-size: 1.5rem;
    }
    
    @media only screen and (max-width:50em){
        header .menu .one .first:nth-child(2),
        header .menu .one .first:nth-child(3)
    {
            display: none;
    }
        header .menu .one{
            font-size:1.2rem;
        }
    }
    @media only screen and (max-width:30em){
        .article .our{
            margin-bottom: 2rem;
        }
        header .menu .one{
            font-size:1rem;
        }
    }

        上面这一小段代码是我网页头部的响应式，没上面技巧吧，也就是平时写静态网页的方法。为了能适应屏
    幕需要把头部的一些li去掉,当然在你去掉的时候要选择一些不那么主要的删除。
        在这里用到了结构伪类选择器的nth-child()；表示选择第几个li，当在两个li中间还有其它标签的时候，同
    样要计算里面。若要选择同类型且不计算不同的标签则可以用nth-of-type(),表示只选择同类型的标签，当在两
    个相同标签之间有不同标签时不计算。

        响应式图片
        图片在响应式里面也是一个大问题，大图片在加载上回很慢，很费流量，而小图片虽然加载速度快但是在
    放大后就会出现模糊，因此就有了响应式图片。响应式图片不仅仅是图片的排版和布局，更多的根据设备大小
    加载不同的图片


        几种实现响应式图片的方式


        1. js或服务端
        2. srcset
        3. srcset配合sizes
        4. picture
        5. svg

        srcset：
        srcset是img标签里的一个属性，看这段代码
        <img src="./img/photo2.png" srcset = "img/480.png 480w, img/800.png 800w,img/1600.png 1600w">
    上面写的图片是一样的，只是大小不一样。添加srcset之后图片会随着屏幕的大小来选择合适的图片来显示，
    在图片地址后面的…w表示图片的宽度。

        srcset配合sizes：
        <img src="./img/photo2.png" srcset = "img/480.png 480w, img/800.png 800w,img/1600.png 1600w"
    sizes="100vw">
        在上一个里没有写sizes属性，但是它其实还是被默认设置为sizes=100vw,这里的vw表示的就是视口的宽
    度即viewport和width,而100表示的是百分比100％而不是一100个，设置成100vw就表示把图片设置成100%
    的视口宽度。sizes属性就是告诉浏览器以什么样的尺寸来显示。

        picture:
    <picture>
        <source srcset = "./img/ad002-l.png" media = "(min-width:50em)">
        <source srcset = "./img/ad002-m.png" media = "(min-width:30em)">
        <img src="./img/ad003.png" >
    </picture>
        picture里包含任意个数的source元素和一个img。而每一个都可以给它设置media(媒体查询)。在这里浏览
    器会遍历picture里面的suorce设置，直到找到一个属性满足当前环境的media，如果满足就把source里的srcset
    匹配到img里面。也就是说我们可以给img匹配多组的srcset属性来匹配不同的媒体查询条件，不像前面那个只
    能设置一组。

        svg：
        svg是可缩放的矢量图形它是基于可扩展标记语言来生成的，它可以用任何的文本编辑器来创建，但是这会
    非常麻烦。这里有个概念——矢量图形，什么是矢量图形？也就是这些图形在缩放的时候都不会失真和变形，这
    些图形不是基于像素的，而是基于有个绘制的规则，比如说颜色，形状轮廓，大小等等，确定了这些规则，即使
    我们发生缩放也不会影响图片的绘制，当然也就不会发生失真。


        创建svg图片网站和查看兼容性：
    1. https://icomoon.io/
    2. https://editor.method.ac/
    3. https://caniuse.com/　（查看兼容性）


        兼容性
        在这里我只说写picture的兼容性，因为我只写了这个。一般在写picture时会使用一个polyfill，polyfill是
    让浏览器能够兼容picture，它是一个脚本。而最有名的polyfill就是picture库。
        picture库的地址：http://scottjehl.github.io/picturefill/，在写的时候直接下载picture库引入就可以了。
